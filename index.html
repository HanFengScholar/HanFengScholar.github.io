1. Windows

1.1 SID为操作系统的身份证明

Administrators组账户的SID为S-1-5-32-544，由4个“-”符号将整个SID分隔为5
组字符和数字，其中:

字母S表示该串字符是一个SID;
数字1表示SID的修订级别;
数字5表示SID的颁发机构，5代表NTAuthority;
数字32表示SID的域标识符，32代表“内置”;
数字544表示SID的相对标识符，544代表Administrators。

由用户创建的用户账户的SID的组成结构与系统内置账户的SID类似，但是SID的长
度通常会更长。例如，下面就是一个由用户创建的用户账户的SID。

S-1-5-21-2319884204-2662211735-595857245-1001

1.6 端口

  • 周知端口(Well Known Ports)

周知端口是众所周知的端口号，范围从0到1023,其中80端口分配给ww服务,21端口分配
给FTP服务等。我们在IE的地址栏里输入一个网址的时候(比如www.baidu.com)是不必指定
端口号的，因为在默认情况下www服务的端口号是“80”

  • 动态端口(Dynamic Ports)

动态端口的范围是从1024到65535之所以称为动态端口,，是因为它一般不固定分配某种
服务，而是动态分配。动态分配指当一个系统进程或应用程序进程需要网络通信时，它向主
机申请一个端口，主机从可用的端口号中分配一个供它使用。

  • 使用netstat命令查看端口状态情况，该命令与Linux类似
选项 含义
-a 显示所有活动的tcp连接，以及计算机监听的tcp和udp端口
-e 显示以太网发送和接收的字节数、数字包数等统计信息。
-n 网络IP地址代替名称，显示出网络连接情形，显示所有已建立的有效连接
-o 显示活动的tcp连接并包括每个连接的进程pid。
-s 按协议显示各种连接的统计信息，包括端口号，此选项可以与-s选项结合使用。
-r 显示核心路由表，格式同“route-e”。本选项可以显示关于路由表的信息。
-p 显示指定协议信息。显示协议名查看某协议使用情况

2. Linux

2.1 /etc/passwd

/etc/passwd文件是UNIX安全的关键文件之一。该文件用于用户登录时校验用户的登录名(LOGNAME)、加密的口令数据项(PASSWORD)、用户ID(UID)、默认的用户分组ID(GID)用户信息(USERINFO)、用户登录子目录以及登录后使用的She11(SHELL)。这个文件的每一行保存一个用户的资料，而用户资料的每一个数据项采用冒号“:”分隔，

在安全检查中需要着重注意该文件的权限，默认情况下/etc/passwd权限为0644。使用命令“stat /etc/passwd”进行检查，如果该文件的权限和属主发生了变化，则可能代表发生了异常情况(例如误操作或入侵事件)，需引起注意。

2.2 /etc/shadow

Linux/UNIX广泛采用了“shadow(影子)文件”机制，将加密的口令转移到/etc/shadow文
件里，该文件只为root超级用户可读，而同时/etc/passwd文件的密文域显示为一个x，从
而最大限度地减少了密文泄露的机会。

域名 含义
username 用户登录名
passwd 加密的用户口令
lastchg 最后一次修改时间:表示从1970年1月1日起到上次修改口令所经过的天数
min 最小修改时间间隔:表示两次修改口令之间至少经过的天数
max 密码有效期:表示口令还会有效的最大天数，如果是99999则表示永不过期
warn 密码需要更改前警告天数:表示口令失效前多少天内系统向用户发出警告
inactive 密码过期后的宽限天数
expire 账户失效时间:表示用户被禁止登录的时间
flag 保留域，暂未使用

2.3 /etc/group

/etc/passwd文件中包含着每个用户默认的分组ID(GID)。在/etc/group文件中，这个GID被映射到该用户分组的名称以及同一分组中的其他成员中去。

4.2 系统排查

4.2.1 win+r 输入 msinfo32

在进行系统基础排查时，可使用微软系统信息工具msinfo32查看本地服务器硬件资源、组件和软件环境信息，可对正在运行的任务、服务、系统驱动和加载模块进行排查

显示本地计算机及其操作系统的详细配置信息，包括:操作系统配置、安全信息、产品ID和硬件属性，如RAM、磁盘空间、网卡和KB补丁信息等

4.2.3 账号排查

在服务器被入侵后，攻击者会建立相关账号(有时是隐藏或者克隆账号)，方便进行远程控制。攻击者会直接建立新账号或者激活一个系统中默认账号或者建立隐藏账号(账号名称后加$符号)。无论攻击者采用哪种方法，都会在获取账号后，使用工具或直接利用相关漏洞通过这个账号进行权限提升,控制服务器。
使用命令查看恶意账号时无法直接查看到隐藏账号

  • win+r 输入lusrmgr.msc采用图形化方式查看是否有新增或者可疑、隐藏账号

  • 打开注册表编辑器，在注册表HKEY_LOCAL_MACHINE\SAM\SAM\Domains\Account\Users\Names，分别查看administrator的F值与存在$账号的F比较，相同则存在克隆账号

  • 在命令行中输入wmic useraccount get name,SID命令，可以直接查看系统中所有用户信息。

4.2.4 启动项

启动项是开机时系统在前台或者后台运行的程序。操作系统在启动时，通常会加载很多程序，启动项
是病毒后门等实现持久化驻留的一种常用方法，在应急响应中排查的必要项目。在Windowx系统中自
启动的文件是按照2个文件夹和5个核心注册表的子健来自启动加载程序的。

  • win+r，输入msconfig系统配置查看

  • 通过注册表查看
    注册表是操作系统中重要的数据库，用于存储系统所必需的信息。以分层的组织形式存储数据元素。数据项是注册表的基本元素，每个数据项下面不但可以存储很多子数据项，还可以以键值对的形式存储数据。很多病毒木马通过注崩表来实现在系统重持久化驻留。
    Windows注册表标块以下5个根键:

    1. HKEY_CLASSES_ROOT:含了文件类型和关联程序的信息，它决定了系统中每个文件类型的默认打开方式
    2. HKEY.CURRENT USER:含了当前用户的配置信息，包括京面设置、文件夹还项、应用程序设置等
    3. HKEV_LOCAL_MACHINE:包含了系统的硬件和软件配置信息，包括安装的设备驱动程序，系统服务和安装的应用程序
    4. HKEV CURRENT CONFIG:包含了当前系统的硬件配置信息，包括显示器、键盘、标等设备的配置信息
    5. HKEY USERS:含了所有用户的配置信息，包括当前用户和其他用户的配置信息。
    6. HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Run 意看是否存在病击、木马启动项

4.2.5 计划任务

由于计算机都会自动加载“计划任务”，“计划任务”也是恶意病毒实现持久化驻留的一种常用手段也是应急响应事件重点排查的项目之一.

  • “计算机管理”-“系统工具”-“任务计划程序”-“任务计划程序库”选项
  • 在PowerShell中输入Get-ScheduledTask也可查看任务计划
  • 在CMD中输入schtasks，也可查看任务计划

4.3 进程查看

进程(Process)是计算机程序关于某数据计划上的一次运行活动，是系统进行资源分配和调度的基本单元，是操作系统结构的基础。在Windows系统中，主机在感染恶意程序后，恶意程序都启动相应的基础来完成相关恶意的操作，有的恶意进程为了能够不被查杀，还会启动相应的守护进程来对恶意进程进行守护。

  • 通过“任务管理器”查看名称为“iexplorer”的可疑进程

  • tasklist 显示本机的所有进程，显示信息包括:进程名(lmage Name)、PID、会话名
    (Session Name)、会话(Session)、内存使用(Mem Usage)情况

  • tasklist /m shell32.dll 查看所有调用shell32.dll动态链接库的进程情况

    shell32.dll是Windows的32位外壳动态链接库文件，用于打开网页和文件，建立文件时的默认文件名的设置等大量功能。系统文件shell32.dll出错，极有可能是盗号木马、流氓软件等恶意程序所导致，其感染相关文件并加载起来，一旦杀毒软件删除被感染的文件，就会导致相关组件缺失

  • tasklist /m rundll32.dll /s 192.168.1.6 /u administrator /p root 查看远程主机调用的rundll32.dI情况

  • 火绒剑查看进程信息,一款用于分析、处理恶意程序的安全工具软件

  • 使用wmic进程排查

    WMIC扩展WMl(Windows Management Instrumentation，Windows管理工具)，提供了从命令行接口和批命令脚本执行系统管理的支持。在命令行中使用wmic process get name,parentprocessid,processid /format:csv 命令，以csv格式显示进程名称、父进程ID、进程ID:

4.4 服务查看

服务可以理解为运行在后台的进程。这些服务可以在计算机启动时自动启动，也可以暂停和重新启动而且不显示任何用户界面。服务非常适合在服务器上使用，通常在为了不影响在同一台计算机上工作的其他用户，且需要长时间运行功能时使用。在应急响应排查过程中，服务作为一种运行在后台的进程，也是恶意软件常用的驻留方式。

  • 打开“运行”对话框，输入“services.msc”命令，可打开“服务”窗口，查看所有的服务项，包括服务的名称、描述、状态等，也可以使用火绒剑

Windows很多第三方程序时通过某些服务运行，如果攻击者获取了第三方程序名称，即可利用相关漏洞进程提权
服务信息的获取，还可用于关闭杀毒软件和防火墙以及关闭某些防护的进程

  • 可通过net start命令，查看系统开启的所有服务
  • 通过net stop service name, 停止服务
  • 通过net start service name, 开启服务

4.5 临时文件

各个盘下的 temp(tmp)相关目录有些恶意程序释放子体(即恶意程序运行时投放出的文件)一般会在程序中写好投放的路径，由于不同系统版本的路径有所差别，但是临时文件的路径相对统一，因此在程序中写好的路径一般是临时目录。对敏感目录进行的检查，一般是查看临时目录下是否有异常文件

  • C:\Users\ADMINI~1\AppData\Local\Temp 为临时文件路径
  • win+r输入%temp%

4625 用户登录失败 大部分登录事件失败时会产生的日志(解锁屏幕并不会产生这个日志)
4672 特殊权限用户登录 特殊权限用户录成功时会产生的日志，例如，登录Administrator，一般会看到4624和4672 日志一起出现
4648 显式凭证登录 一些其他的登录情况，如使用runas /user 以其他用户身份运行程序时会产生的日志(不过在使用runas时，也会产生一条4624日志)

ls -l /tmp/

/tmp目录通常用于存储临时文件权限通常设置为777，任何用户都可以访问和修改该目录中的文件攻击者会将可执行的木马、病毒下载至该目录中，以便执行。

~/.ssh和/etc/ssh

~/.ssh和/etc/ssh存放密钥的目录也作为一些后门程序的保存的路径，攻击者有时会利用公钥泄露实现免密登录。

文件时间点排查

通过列出攻击日期内变动的文件，可发现相关恶意文件，通过find命令可以对某一个具体时间段内新增的文件查找。

webshell排查

Webshell是一种以.asp、.php、.jsp或.cgi等网页文件形式存在的一种代码执行环境，也可以理解为一种网页后门。常见的webshell有一句话木马、小马和大马三种。一句话木马，代码量小，功能少，不易被发现，需要专业查杀工具排查。Webshell的排查可以通过分析文件、流量、日志进行，基于文件命名特征和内容特征，操作性高。如果是通过文件方法排查，可以从webshell常出现的关键字入手，对文件进行初筛，缩小排查范围，即可使用之前学习的find命令进行搜索，或者使用专业查杀工具(D盾、河马等)进行排查。
find -name “*.php” | xargs egrep POST/eval/cmd # 查看以php结尾内容包含POST/eval/cmd的文件
河马：https://www.shellpub.com/

文件完整性校验

  • 通过与原始代码md5值对比差异，若被篡改，则md5值不同例如：md5sum muma1.txt
  • 也可使用diff命令查看文件内容差异，判断文件是否被修改例如：diff -c -a -r muma1.txt muma2.txt

后门排查

攻击者会在目标对象上安装rookit后门程序，该程序非常隐蔽、不易被察觉，被植入后为攻击者建立一条长时间入侵、实时监控和控制的途径导致系统被控制。针对此类后门程序

可使用第三方的查杀工具如chkrootkit、rkhunter进行检测

suid

SUID是Linux中的一种特殊权限，功能为用户运行某个程序时，如果该程序有SUID权限，那么程序运行进程时，权限为该文件的属主权限。例如程序文件属主为root，那么执行程序的用户暂时拥有root权限。

5.6 流量分析

一般使用第三方工具或者netstat查看相关地址连接情况，但是内部流量无法查看。
如果要查看其内部流量情况，需要对网络通信抓包分析，并对数据报文过滤分析。
Wireshark可以用于捕获和分析网络流量的工具
利用wireshark抓包查看攻击流量并分析这些攻击流量的特征，识别并有效阻止进一步的攻击行为。

由于勒索病毒WannaCry是利用Windows的smb协议进行攻击，可以通过wireshark对“永恒之蓝漏洞攻击流量进行分析，获取整个攻击流量的通信过程。在选器中输入smb，只查看smb协议流量攻击载荷发送NT Trans Request载荷，包含大量NOP指令，再次发送Trans2 Secondary Request载荷，分几个数据包发送加密的攻击报文

6. 溯源实战

6.1 攻击源捕获

在进行溯源反制前，需要确定攻击源，可以从安全设备、日志审计、服务器器异常、邮件钓鱼、蜜罐系统进行捕获。
攻击源捕获方式:

  • 安全设备辅助：利用IDS/IPS/态势感知/蜜罐/沙箱/全流量设备高级辅助获取初步攻击IP、攻击手法、恶意样本信息。
  • 日志审计与流量分析：通过异常流量分析攻击行为、攻击手法、攻击源地址与目的地址。
  • 服务器异常捕获：异常文件程序、非正常用户、恶意外部通道、诡异进程、端口、计划任务、启动项、注册表、快捷方式以及隐藏文件。

6.2 溯源反制

IP定位:根据IP定位物理地址:通过IP端口扫描，反向渗透服务器进行分析，最终定位攻击者相关信息。
ID追踪:根据搜索引擎、社交平台、技术论坛、社工库匹配:利用ID从技术论坛追溯邮箱，继续通过邮箱反追踪真实姓名，通过姓名找到相关简历信息
网站URL:域名Whois查询注册人姓名、地址、电话和邮箱:通过攻击IP历史解析记录/域名，对域名名注册信息进行溯源分析
恶意样本:提取样本特征，如用户名、ID、邮箱、C2服务器:在样本分析过程中，发现攻击者主机信息浏览器信息、真实IP及社交信息。
封禁IP后，将该IP地址上传至威胁情报平台，被平台判定为恶意IP，再进行溯源反制
